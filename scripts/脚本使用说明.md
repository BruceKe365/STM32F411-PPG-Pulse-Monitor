# Python 脚本使用说明

本目录集中存放项目的 Python 辅助脚本，包括串口采集、数据下载、模型训练、MCU 等价验证和文档生成。

所有命令建议在项目根目录，也就是 clone 后的仓库根目录下运行：

```powershell
python scripts/脚本名.py
```

不要先 `cd scripts` 再运行。多数脚本默认使用项目根目录下的 `training_dataset/`、`testing dataset/`、`Core/Src/main.c` 和 `Reports/` 等相对路径。

## 脚本总览

| 脚本 | 用途 |
| --- | --- |
| `scripts/read_serial_diagnostics.py` | 读取 STM32 USB CDC 诊断串口输出，用于检查主循环、采样、OLED、CDC 是否卡顿。 |
| `scripts/read_max30102_raw.py` | 采集 MAX30102 red/ir 原始数据，也支持离线回放、处理、生成 processed/summary 文件。 |
| `scripts/download_af_training_data.py` | 下载 PhysioNet AF/NSR 相关公开数据的最小注释文件。 |
| `scripts/train_af_naive_bayes.py` | 从 RR/PPI 窗口提取 AF 特征，训练离散朴素贝叶斯模型，导出 JSON 和 MCU 头文件。 |
| `scripts/simulate_af_ppi_risk.py` | 生成模拟 PPI 场景，检查 AF 模型输出是否符合直觉。 |
| `scripts/validate_mcu_af_live.py` | 复刻 MCU AF live/test 策略，是 AF 功能最重要的电脑端验证脚本。 |
| `scripts/download_stress_training_data.py` | 下载并整理 WESAD 压力训练数据。 |
| `scripts/train_stress_hrv_model.py` | 提取 HRV 特征并训练压力 Logistic Regression 模型，导出 JSON 和 MCU 头文件。 |
| `scripts/validate_stress_hrv_live.py` | 复刻 MCU Stress live 显示、质量门控和 high HR 策略。 |
| `scripts/build_pulse_report_docx.py` | 生成 `Reports/脉搏测试仪项目补充汇报.docx` 的文档辅助脚本。 |

## 常用命令

串口诊断：

```powershell
python scripts/read_serial_diagnostics.py
```

采集 MAX30102 原始数据：

```powershell
python scripts/read_max30102_raw.py --port COM7 --duration 20 --csv ppg_raw_test.csv
```

训练 AF 模型：

```powershell
python scripts/download_af_training_data.py --datasets afdb ltafdb nsr2db
python scripts/train_af_naive_bayes.py
```

验证 MCU 等价 AF 逻辑：

```powershell
python scripts/validate_mcu_af_live.py
```

训练压力 HRV 模型：

```powershell
python scripts/download_stress_training_data.py
python scripts/train_stress_hrv_model.py --wesad-dir training_dataset/wesad --local-ppg-glob "testing dataset/**/*processed*.csv"
```

验证 MCU 等价 Stress live 逻辑：

```powershell
python scripts/validate_stress_hrv_live.py --out-dir training_dataset/reports/full_stress_live_validation_YYYYMMDD_candidate --ppg-glob "testing dataset/**/*processed*.csv" --ppg-group all_local_processed --step-s 60
```

生成补充汇报 Word 文档：

```powershell
python scripts/build_pulse_report_docx.py
```

## 注意事项

- 脚本移动到 `scripts/` 后，项目根目录不再直接放置 `.py` 文件。
- 训练和验证脚本默认从项目根目录解析数据路径，命令示例都按根目录运行编写。
- `training_dataset/models/` 是当前正式模型参数目录，不要随手覆盖；新实验建议输出到带日期的 candidate 目录。
- `training_dataset/physionet/` 的 AF 最小注释文件已随当前提交保留；缺失时用 `download_af_training_data.py` 重新下载。
- `training_dataset/wesad/` 和 `training_dataset/derived/` 不随 Git 提交；只有重新训练 Stress 模型时才需要下载/生成。
- `testing dataset/` 是本地 MAX30102 采集和回放数据，不是真实 AF 阳性训练集。
- AF risk 和 Stress index 都是课程设计中的风险提示/指数原型，不是医学诊断。

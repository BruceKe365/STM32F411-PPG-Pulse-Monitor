## 一句话状态

压力/情绪功能已经完成“WESAD 训练 + 本地 PPG 验证 + MCU 端移植 + STRESS 页面显示”，当前为验收定稿实现。它不是识别开心/难过，而是基于 PPG 峰间期 PPI 的 HRV 压力指数，输出 1-99 数值和 `relax/normal/medium/high` 四档。

## 算法路线

```text
MAX30102 PPG -> robust peak -> PPI -> HRV 特征 -> Logistic Regression -> stress probability -> 1-99 stress index
```

核心点：

- 设备只有 MAX30102 PPG，因此 MCU 端只使用 PPI/HRV 特征。
- 训练用 WESAD，但默认取 wrist BVP 信号，不依赖 EDA、呼吸、体温或 ECG 形态。
- MCU 端不训练模型，只做实时 PPI 收集、HRV 特征计算、标准化、线性加权、sigmoid 和指数映射。
- 压力值容易受高心率、运动、手指抖动和漏峰影响；当前通过显示门控降低误解风险。

## 当前 MCU 行为

相关代码集中在 `Core/Src/main.c`：

```text
STRESS_HRV_MIN_INTERVAL_COUNT = 28
STRESS_HRV_WINDOW_TARGET_MS = 40000
STRESS_HRV_FIRST_STEP_MS = 10000
STRESS_HRV_REFRESH_STEP_MS = 30000
STRESS_HRV_HIGH_HR_BPM = 120
```

实时计算条件：

```text
finger_present != 0
ppg_state.raw_finger_present != 0
ppg_state.finger_score >= PPG_MIN_SPO2_SCORE
ppg_signal_valid != 0
heart_rate_bpm > 0
spo2_percent > 0
```

PPI 来源：

- 复用 PPG robust peak。
- 只有 HR/SpO2 有效时才记录 PPI。
- PPI 会经过 HR 参考的动态区间过滤：
  - 正常 HR：`0.70x ~ 1.45x` 当前期望 PPI。
  - HR < 65：`0.58x ~ 1.65x`。
  - HR > 115：`0.64x ~ 1.38x`。
- 每个更新周期内，如果 PPI rejected 比例超过 20%，不更新压力值，保持旧显示。

窗口和刷新：

- 需要至少 28 个 PPI。
- 最近窗口覆盖约 40 秒。
- 首次未出值时，每 10 秒尝试计算一次。
- 出现第一个可显示压力值后，每 30 秒尝试更新一次。
- 当 `heart_rate_bpm > 120` 时，STRESS 页面第二行显示 `-- high HR`，后台仍继续计算，但刷新节奏按“暂无可参考显示值”走 10 秒。

显示：

```text
第一行：STRESS
第二行：数值 + 英文档位，或 -- high HR
```

档位：

```text
1-29   relax
30-59  normal
60-79  medium
80-99  high
```

## 模型和脚本

| 文件 | 用途 |
| --- | --- |
| `scripts/download_stress_training_data.py` | 下载/整理 WESAD 数据。 |
| `scripts/train_stress_hrv_model.py` | 从 WESAD 训练压力 HRV Logistic Regression，并生成模型/报告。 |
| `scripts/validate_stress_hrv_live.py` | 用本地 processed PPG 复刻 MCU stress live 策略，验证显示门控和本地数值范围。 |

注意：正式模型文件在 `training_dataset/models/`。如果只是测试新训练流程，不要直接覆盖正式模型；可以使用隔离 `--base-dir` 生成到 `training_dataset/reports/...` 下。

## 数据集

| 数据 | 路径 | 角色 |
| --- | --- | --- |
| WESAD | `training_dataset/wesad/` | 压力模型训练/验证。当前默认使用 wrist BVP。 |
| 本地 MAX30102 PPG | `testing dataset/**/*processed*.csv` | 正常/边界烟测，验证 MCU live 显示策略和质量门控。 |

WESAD 是公开压力数据集，但它不是本设备采集的数据；本地 PPG 没有真实心理压力标签，因此本地数据主要用于检查“静息正常时是否显示合理”和“坏 PPI 是否被门控”。

## 修改边界

- 不要为了压力功能修改 HR/SpO2 主算法。
- 不要直接把高心率解释成心理压力；当前策略是 HR>120 时显示 `-- high HR`。
- 不要把本地正常 PPG 当成有压力标签的训练数据。
- 不要轻易缩短 40s 窗口或 28 个 PPI 门槛；压力 HRV 对假峰很敏感。
- 如果要改模型，先在 PC 端完整跑 WESAD 和本地 processed PPG，再考虑移植 MCU。

## 展示口径

建议对外表述：

> 当前压力/情绪功能是基于 PPG 心率变异性 HRV 的端侧压力指数原型。模型使用 WESAD 的 BVP 数据训练，单片机端只使用 MAX30102 可得到的 PPI 间期特征做轻量 Logistic Regression 推理，输出 1-99 压力指数和 relax/normal/medium/high 四档。高心率时显示 `-- high HR`，避免把运动或非静息状态误判为心理压力。

# 公开训练数据、模型与验证报告说明

更新时间：2026-06-24

本目录保存 AF 风险和压力/情绪功能相关的公开数据集、派生训练窗口、模型参数和验证报告。它和 `testing dataset/` 不同：这里主要是公开 RR/PPI、WESAD、训练模型和验证报告；`testing dataset/` 是本地 MAX30102 PPG 采样与 HR/SpO2/OLED/AF/压力回放数据。

## 2026-06-24 文档命名说明

本说明文件由 `training_dataset/README.md` 改名为 `training_dataset/公开训练集说明.md`，方便人工和 AI 直接识别用途。`physionet/`、`wesad/`、`derived/`、`models/`、`reports/` 等目录名保持英文，因为训练和验证脚本默认读取这些路径。

## 2026-06-23 验收数据封版

本目录当前作为验收定稿数据和模型存档使用。2026-06-17 完成最后一次模型、验证报告和本地数据同步；截至 2026-06-23，没有新增训练数据、重新训练模型或生成新的候选报告。

- 不把 `testing dataset/` 的本地正常采集当成 AF 阳性数据或 Stress 标签训练数据。
- 历史报告用于说明演进，当前结论以三个 `full_*_20260629_current` 报告为准。

## 数据集说明

当前公开数据集都在 `physionet/` 下。

| 数据集 | 路径 | 当前用途 | 参与训练 | 标签口径 |
| --- | --- | --- | --- | --- |
| AFDB | `physionet/afdb/1.0.0` | AF/非 AF 训练与验证 | 是 | `.atr` rhythm 给 AF/normal/mixed，`.qrs` 给 beat time。 |
| LTAFDB | `physionet/ltafdb/1.0.0` | 长时 AF/非 AF 训练与验证 | 是 | `.atr` 同时给 beat 和 rhythm。 |
| NSR2DB | `physionet/nsr2db/1.0.0` | 正常窦律训练与验证 | 是 | `.ecg` 给 RR 注释，整库按 normal 处理。 |
| MITDB | `physionet/mitdb/1.0.0` | 额外泛化验证 | 否 | `.atr` 给 beat/rhythm，验证时拆成 AF/normal/other/mixed。 |
| NSRDB | `physionet/nsrdb/1.0.0` | 额外正常泛化验证 | 否 | `.atr` 给 beat，整库按 normal 处理。 |

压力/情绪数据：

| 数据集 | 路径 | 当前用途 | 标签口径 |
| --- | --- | --- | --- |
| WESAD | `wesad/` | 压力逻辑回归训练和按受试者划分验证 | baseline/amusement 作为非压力，stress 作为压力；默认使用腕部 BVP。 |

## Git 提交状态和下载方式

当前公开仓库准备状态如下：

| 数据/产物 | 是否随 Git 提交 | 说明 |
| --- | --- | --- |
| `physionet/` | 是 | 只提交 AF/NSR 训练验证所需的最小注释文件。 |
| `models/` | 是 | 当前 MCU 使用的 AF/Stress 模型参数。 |
| `reports/full_*_20260629_current/` | 是 | 与当前固件参数一致的验证报告。 |
| `wesad/` | 否 | WESAD 原始 zip/解压目录体积约 GB 级，仅在重新训练 Stress 模型时下载。 |
| `derived/` | 否 | 派生训练窗口，可由训练脚本重新生成。 |
| 历史 `reports/*` | 否 | 只保留最终 current 报告，历史实验可按需重跑。 |

如果 `physionet/` 缺失或需要刷新 AF 公开数据：

```powershell
python scripts/download_af_training_data.py --datasets afdb ltafdb nsr2db mitdb nsrdb
```

如果需要重新训练 Stress 模型，先下载 WESAD：

```powershell
python scripts/download_stress_training_data.py
```

然后在隔离目录训练，避免覆盖正式模型：

```powershell
python scripts/train_stress_hrv_model.py --base-dir training_dataset/reports/full_stress_train_validation_YYYYMMDD_candidate --wesad-dir training_dataset/wesad --local-ppg-glob "testing dataset/**/*processed*.csv"
```

重要限制：

- 当前公开 AF 数据主要来自 ECG 标注，但模型只使用 RR/PPI 间期统计特征，不使用 ECG 波形形态。
- 这使模型可以迁移到 MCU 端 PPG 峰间期，但仍存在 ECG RR 到 PPG PPI 的 domain gap。
- 当前没有真实 AF 患者 PPG 数据，因此结论应表述为“疑似房颤风险原型”或“不规则心律提示”，不是医学诊断。

## 标签解释

`scripts/validate_mcu_af_live.py` 的公开数据窗口标签：

| 标签 | 含义 |
| --- | --- |
| `af` | 窗口主要处于 AF rhythm。 |
| `normal` | 窗口主要处于正常或按整库正常处理的数据。 |
| `mixed` | 一个 30 秒窗口内混入多种 rhythm。 |
| `other` | 其他异常节律或非 AF 异常标签。 |

`mixed` 和 `other` 高风险比例高，不代表模型“错把所有正常人判成房颤”，而是说明 PPI 不规则模型会对早搏、其他异常节律和伪差样不规则敏感。

## 常用命令

下载额外公开数据：

```powershell
python scripts/download_af_training_data.py --datasets mitdb nsrdb
```

重新训练：

```powershell
python scripts/train_af_naive_bayes.py
```

跑全部公开数据验证：

```powershell
python scripts/validate_mcu_af_live.py --include-public --public-datasets afdb,ltafdb,nsr2db,mitdb,nsrdb --sim-runs 0 --out-dir training_dataset/reports/mcu_live_af_all_public
```

跑质量门控实验：

```powershell
python scripts/validate_mcu_af_live.py --include-public --public-datasets afdb,ltafdb,nsr2db,mitdb,nsrdb --sim-runs 0 --quality-gate-experiment --out-dir training_dataset/reports/mcu_live_af_gate_experiment
```

跑当前定稿 AF 验证：

```powershell
python scripts/validate_mcu_af_live.py --out-dir training_dataset/reports/full_af_validation_YYYYMMDD_candidate --ppg-glob "testing dataset/**/*processed*.csv" --sim-runs 120 --sim-duration-s 180 --include-public --public-datasets afdb,ltafdb,nsr2db --public-record-limit 0 --quality-gate-experiment
```

跑当前定稿 Stress live 验证：

```powershell
python scripts/validate_stress_hrv_live.py --out-dir training_dataset/reports/full_stress_live_validation_YYYYMMDD_candidate --ppg-glob "testing dataset/**/*processed*.csv" --ppg-group all_local_processed
```

隔离训练/验证压力模型，不覆盖正式模型：

```powershell
python scripts/train_stress_hrv_model.py --base-dir training_dataset/reports/full_stress_train_validation_YYYYMMDD_candidate --wesad-dir training_dataset/wesad --local-ppg-glob "testing dataset/**/*processed*.csv"
```

一致性检查：

```powershell
python scripts/check_firmware_consistency.py
```
